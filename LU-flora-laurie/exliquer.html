Programmation Asynchrone

Callback

Méthode la plus ancienne pour gérer de l'asynchrone en JS.
Un Callback est une fonction passée en paramètre d'une fonction asychrone.
Lorsque la fonction asynchrone a fini de s'éxécuter, elle va transmettre son résultat à la fonction Callback passée en paramètre. La fonction callback sera donc éxécutée de manière asynchrone
Exemple : element.addEventListener('click', function(e) {
// Do something here ...
});
Pour gérer des erreurs dans les callback on place deux paramètre dans la fonction. Le premier est l'erreur et le 2e est la donnée. Le paramètre erreur sert à renvoyer un message d'erreur.

Promesses

En programmation, les notions de futurs (« futures »), promesses (« promises ») ou délais (« delay ») font référence à des techniques de synchronisation pour certains langages concurrents. Il s'agit d'abstractions qui servent de proxy pour un résultat non-connu au moment où il est référencé pour la première fois, car son calcul ou son obtention se feront « plus tard » à l'exécution.
JS est synchrone et n'a qu'un thread d'éxécution
L'objet Promise (pour « promesse ») est utilisé pour réaliser des traitements de façon asynchrone. Une promesse représente une valeur qui peut être disponible maintenant, dans le futur voire jamais.
Prévoir un try catch pour débuguer les promesses lorqu'on utilise async/await try catch
L'objet promise contient la fonction then() (éxécution du code lorque la promesse est résolue) et la fonction catch() (éxécution du code en cas d'erreur).
L'avantage des promise est qu'elle peuvent être chaînées.
Exemple :
const promise1 = new Promise((resolve, reject) => {
setTimeout(() => {
resolve('foo');
}, 300);
});
promise1.then((value) => {
console.log(value);
// expected output: "foo"
});
console.log(promise1);
/ / expected output: [object Promise]

This

This est un mot clef renvoyant l'adresse de l'objet en cours, c'est à dire la référence sur l'instance courante.

Bind

Méthode JS utilisée pour relier le this d'une fonction callback à la fonction principale.

Requête

Une requête est l'interrogation vers une base de donnée. Le langage le plus connu pour gérer les données est SQL.

Évènement

Un évènement est une réaction émise suite à l'action d'un utilisateur. Un évènement contient une fonction callback.

Listener

On écoute un évènement lorsque celui-ci se déclenche.
La fonction addEventListener(event, callback) fait cela.
